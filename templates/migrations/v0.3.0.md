# Migration: v0.3.0

## Summary
- **Release class:** minor (semver compatible). No breaking changes; no profile/ownership-class change.
- **Upgrade path:** binary self-update, then re-render the corpus. Fully automatic — no manual file edits required for the default case.
- **Risk:** low. Three consumer-visible changes, all non-breaking (one is a permission tightening toward the already-documented W1 model):
  - **Managed-drift honors ownership overrides.** A path raised to `project_owned` via `.vh-agent-harness/harness-ownership.yml` that intentionally diverges from the render is no longer a perpetual `drifted` FAIL — it surfaces as a non-failing `preserved` (INFO) outcome. `doctor` output gains an INFO tier; `preflight` stays exit 0 and never blocks install/update on a preserved file.
  - **shell-guard normalizes git global flags.** Read-only git commands prefixed with execution-safe global flags (`git --no-pager log`, `git -C <repo-root> diff`) are rewritten in-place before the permission matcher sees them, so the existing `git <verb> *` allow rules match and no operator prompt fires. Mutations are still denied regardless of how many flags prefix the verb.
  - **W1 backlog-edit deny for worker agents.** `build` and `docs-steward` can no longer directly edit `docs/planning/backlog.md` — status flows through the single-writer-promotion transport (`/task-update`, `/task-closeout`); only the promoter writes the canonical backlog. AGENTS.md was reconciled to this model in the same arc.

## What changed (consumer-visible only)

The common no-override install is byte-identical (preserved=0 → tierPass). A raised path that is missing from disk is silent (update never seeds raised paths). | `self-update` (binary: `internal/cli/doctor.go`, `internal/cli/preflight.go`) | non-breaking (additive INFO tier; no exit-code change) |
| shell-guard git global-flag normalization | A generic git global-flag registry and walker (`walkGitGlobals` in `shell-guard-core.js`) consumes leading git global flags and rewrites the command in-place in the `tool.execute.before` hook, so opencode's path-blind permission matcher sees the bare `git <verb>` form and matches the existing `git <verb> *` allow rules — eliminating spurious prompts for `git --no-pager log`, `git -C <repo-root> diff`, etc. Flag policies: **always-stripped** display-only flags (`-p`, `--paginate`, `-P`, `--no-pager`); **conditionally stripped** `-C <path>` only when absolute and identical to the command's working directory; **never stripped** behavior-changing flags (`-c`, `--git-dir`, `--work-tree`, `--namespace`, `--config-env`, `--exec-path`, `--no-replace-objects`, `--bare`) and any unknown flag → the rewrite blocks → opencode prompts; **info flags** (`--help`, `--version`, `--html-path`, …) with no verb → allowed directly. A mutation-slip guard extracts the verb past any run of leading globals, so `git --no-pager commit`, `git -C <ext> commit`, and `git -C <repo-root> commit` all DENY. Also fixes a pre-existing bug: `--paging=no` was never a real git flag (absent from `git --help`) and is removed; the canonical always-strip set is now `-p`/`--paginate`/`-P`/`--no-pager`. `GIT_MUTATION_VERBS` is extracted as the single source of truth (the `git-mutation-bypass` regex is now built from it). | `update` (core templates: `shell-guard-core.js`, `shell-guard.js`, `shell-guard/eval.js`, `forbidden-patterns.core.js`) + binary | non-breaking (prompt reduction + mutation-slip closer) |
| W1 backlog-edit deny (build/docs-steward) | `build` and `docs-steward` gain an object-form edit decision `{"*":"allow","docs/planning/backlog.md":"deny"}` (findLast semantics: broad allow first, narrow deny last). Workers can no longer directly edit the canonical `docs/planning/backlog.md` — status intents route through `.local/coordinator/tasks/` transport via `/task-update` (or `/write-task`) and `/task-closeout`; only the promoter batch-promotes into the canonical backlog each cycle. The pattern is the inverse of the committer's broad-deny + narrow-allow. Every other agent's edit decision is unchanged. AGENTS.md was reconciled to the W1 single-writer-promotion model in the same arc. | `update` (permconfig emit → `opencode.jsonc` edit block) + binary | non-breaking (permission tightening toward the documented model) |

## How to migrate (automated)
```bash
vh-agent-harness self-update            # pull the new binary (v0.3.0)
vh-agent-harness version                # expect: 0.3.0 (<label>)
vh-agent-harness update --dry-run       # ownership-safe preview; watch the shell-guard + permconfig renders
vh-agent-harness update                 # applies platform_managed + active overlay_extension; ships the rewrite + W1 deny
vh-agent-harness doctor                 # lint the result (an INFO row may appear if you hold ownership overrides)
```
No profile, ownership, or capability change is required. The drift INFO tier is delivered by the binary alone (`self-update`); the shell-guard rewrite and the W1 edit-deny are delivered by re-rendering the corpus. Consumers with a `local_adopted` customization of `shell-guard-core.js` should review the courtesy-check diff and re-merge onto the canonical form (see Watch-outs).

**What you'll see (drift):** if you hold no `.vh-agent-harness/harness-ownership.yml` overrides, `doctor` output is unchanged. If you have raised a managed path to `project_owned` and it diverges from the render, the `managed-drift` row now reads `INFO … N managed file(s) in sync; M project-preserved (ownership override)` instead of `FAIL … drifted`; `doctor` still exits 0 (HEALTHY) and `preflight` still passes (exit 0). The INFO tier neither fails nor warns.

## What `update` handles for you
- Re-renders `.opencode/plugins/shell-guard-core.js` (plus the `shell-guard.js` shim and `shell-guard/eval.js` bridge) with `walkGitGlobals`, the flag registry, the mutation-slip guard, and the corrected always-strip set (`-p`/`--paginate`/`-P`/`--no-pager`).
- Re-renders `.opencode/repo-configs/forbidden-patterns.core.js` with `GIT_MUTATION_VERBS` as the single source of truth (the `git-mutation-bypass` regex is built from it).
- Emits the `build` and `docs-steward` object-form edit (`{"*":"allow","docs/planning/backlog.md":"deny"}`) into `opencode.jsonc`; every other agent's edit decision is byte-identical to before.
- Delivers the managed-drift override-awareness and the `preserved`/INFO tier via the binary (no rendered file carries it) — `self-update` alone is sufficient for the drift behavior.
- Reconciles schema on seeded files; preserves everything tagged `local_adopted` / inactive `overlay_extension` unchanged.

## Watch-outs
1. **W1 deny is a permission tightening.** If a `build`/`docs-steward` agent was editing `docs/planning/backlog.md` directly (contrary to the now-reconciled AGENTS.md), it now hits an edit deny. Route status through `/task-update` (or `/write-task`) + `/task-closeout`; the promoter promotes into the canonical backlog. README.agent.md's "Track work" guidance was reconciled in this release to match.
2. **`--paging=no` is gone.** It was never a real git flag (absent from `git --help`) and is removed from the always-strip set. Use `--no-pager` (or `-P`). The v0.2.1 migration note referenced `--paging=no` as a synonym; that reference is superseded — the canonical always-strip set is now `-p`/`--paginate`/`-P`/`--no-pager`.
3. **The `preserved` gate is narrow (F3).** Only a path whose effective class was GENUINELY RAISED by an override (`Origin == OriginOverrideRaise`) counts as preserved. A default-class non-managed divergence (project_owned seeds like `README.md`/`Makefile`, the `platform_armed` profile, `external_generated` recon data) is silently skipped as before — NOT labeled preserved. An absent override file means every path keeps `OriginDefault` and behavior is byte-identical to the raw-class check.
4. **`preserved` is informational only.** It does not change `doctor`'s exit code (INFO neither fails nor warns) and never blocks `preflight` (exit 0). A raised-to-`project_owned` path that is MISSING from disk is silent — not drift, not preserved.
5. **Customized shell-guard.** If a previous `local_adopted` edit customized `.opencode/plugins/shell-guard-core.js`, the migration courtesy check fires and `update` will NOT silently clobber it. Review the diff and re-merge your customization onto the canonical form so the flag-registry rewrite and mutation-slip guard both land.

## Rollback
A minor release is reversible by binary downgrade: reinstall the v0.2.2 binary and `vh-agent-harness update` re-renders the v0.2.2 corpus (the render is authoritative for `platform_managed` files, so the shell-guard rewrite, the W1 edit-deny, and the `preserved`/INFO tier all revert to their v0.2.2 form). Rolling back reverts the prompt reduction (more operator prompts for global-flag-prefixed git reads — annoying, not a security hole: the v0.2.1 `git --no-pager <mutation>` closer still holds), re-opens direct backlog edits for workers (contrary to the reconciled AGENTS.md — prefer forward-fixing instead), and re-introduces the perpetual `drifted` FAIL on intentionally-overridden paths. No data migration ran, so there is nothing to undo beyond the binary reinstall and re-render. A `local_adopted` customization survives the rollback (it is preserved in both directions):
```bash
git checkout HEAD -- .opencode/         # only if an unrelated render diff is unwanted
```
