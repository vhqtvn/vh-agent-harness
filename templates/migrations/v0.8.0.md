# Migration: v0.8.0

## Summary
- **Release class:** minor (semver compatible). No breaking changes; no profile/ownership-class change. The `v0.7.0..HEAD` arc carries 1 `feat:` commit (`40eed7d`, debate auto reframe-and-diverge Layer 2) and 1 `fix:` commit (`f0bd77e`, unified frame-element vocabulary) with **zero `!:` subjects and no `BREAKING CHANGE:` footers** → MINOR bump; not v1.0.0. Last tag = `v0.7.0`; intended version = `v0.8.0`.
- **Upgrade path:** binary self-update, then re-render the corpus. Fully automatic for the default consumer — the entire arc is an additive refinement of the debate capability that ships through `go:embed` + `vh-agent-harness update`.
- **Risk:** low. The arc is debate-only and purely additive: a new bounded debate mode (auto reframe-and-diverge), a new typed critic field (`frame_level_trigger`), and a unified closed frame-element vocabulary. The default debate flow is unchanged; the new behavior fires only when a critic emits a validated frame-level trigger.

The central thing to understand about this release: **the debate capability gains a bounded auto reframe-and-diverge step (O5 Layer 2) plus a unified frame-element vocabulary.** Layer 2 is a single bounded event the orchestrator may authorize ONLY after a critic `frame_level_trigger` clears five conjunctive conditions. It does not broaden into brainstorming and does not extend the revision budget. The follow-up fix (`f0bd77e`) closes two contract findings from the Layer 2 review: the critic's `original_frame_element` and the proposer's `frame_delta` now draw from ONE closed 8-element set, and the proposer's `outside_frame_candidates` are now full option objects (not bare id lists).

## Watch-outs
1. **Reframe bounds are HARD CAPS.** One event, one alternate frame, ≤2 outside-frame candidates, ≤5 total active options, depth 1. A reframe that would breach the 5-option cap must park/drop original options first; the proposer must not exceed the caps.
2. **Reframe consumes the existing revision budget.** There is no free extra revision round. If the revision budget is already spent, a reframe may still produce an alternate frame, but no further revision cycle is granted.
3. **The forbidden-trigger list is exhaustive, not illustrative.** Ties, generic low diversity, low confidence, ordinary disagreement, an unpopular leader, and "the options feel unsatisfying" can NEVER authorize a reframe on their own. Homogeneous-looking options are a diagnostic CLUE only — they may prompt the critic to look harder, but cannot fire a reframe.
4. **Suppress is operator-initiated, not automatic.** The default debate/solution-brief flow does not suppress an auto reframe; the operator must explicitly request it. An auto reframe that the operator disagrees with is retained until suppressed.
5. **The frame vocabulary is CLOSED.** A `frame_delta` must change ≥1 of the eight named elements; an element outside the set is invalid. A trigger whose proposed change touches no dimension is within-frame diversity and is rejected back to ordinary revision (`no_frame_delta`).

## Rollback
A minor release is reversible by binary downgrade: reinstall the `v0.7.0` binary and `vh-agent-harness update` re-renders the `v0.7.0` corpus (the render is authoritative for `platform_managed` files). **Caveats:** rolling back (a) removes the auto reframe-and-diverge Layer 2 policy, the `frame_level_trigger` typed field, the unified frame vocabulary, and the solution-brief suppress clause from the rendered agents; (b) leaves any locally adopted debate/solution-brief customizations in place as `local_adopted` state. No data migration ran, so there is nothing to undo beyond the binary reinstall and re-render. A `local_adopted` customization survives the rollback and triggers the standard courtesy check on the way back up:
```bash
git checkout HEAD -- .opencode/         # only if an unrelated render diff is unwanted
```
