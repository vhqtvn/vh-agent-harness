The central thing to understand about this release: **the auto-classifier safety system ships in two layers.** The **building blocks are in the binary** — the `sys-prompt` CLI, the embedded auto-gate system prompts, and the `doctor` auto-classifier config-shape validator. The **enforcement runtime is opt-in** — the `permission.asked` event hook, the multi-tier consensus, the live LLM adapter (retry-on-idle, v1-on-v2-server-error fallback), the config layers (`defaults <- user <- project`), the dual-form endpoint+key, and the per-call deny gate all live in the **opt-in `auto-classifier-pilot` overlay pack**, which is **embedded in the binary (under `templates/overlays/`) and selectable by name out of the box — nothing to vendor or recreate**. Consumers who want enforcement select it via `overlays: [auto-classifier-pilot]`; consumers who do not opt in get the inert building blocks only.

## Watch-outs
1. **Auto-classifier enforcement is OPT-IN.** The pilot overlay ships **embedded in the binary** under `templates/overlays/` and is name-selectable (it is kept out of `templates/core/` so it stays a self-contained overlay unit, not a default-rendered core file). Default consumers get the inert building blocks (`sys-prompt` CLI, embedded prompts, `doctor` config validator) only — no `permission.asked` hook, no consensus, no live LLM adapter. To enable enforcement, a consumer adds `auto-classifier-pilot` to `overlays:` and re-renders; no vendoring or recreation is needed.
2. **`doctor` `auto-classifier` only validates configs WHEN PRESENT.** A non-opt-in consumer (no overlay) sees nothing. An opt-in consumer with a malformed `auto-gate-config.json` / `auto-gate-llm.json` FAILs `doctor`; a correct one passes. Do not interpret a clean `doctor` on a default install as "the classifier is configured" — it simply means there is nothing to validate.
3. **F-intent transform is project-owned.** If a project maintains a `.mjs` F-intent transform, it runs at render time; a broken `.mjs` breaks the render (it is project-owned, not harness-shipped). Projects without one are unaffected.
4. **Auto-classifier deny is a per-call gate, not a session kill-switch.** A deny blocks that one permission ask, not the whole session (fixed in `db03275`). Operators piloting enforcement should understand this scoping before relying on it.
5. **Dual-form endpoint+key (literal preferred).** The auto-classifier LLM endpoint+key may be supplied in two forms; the literal form is preferred. Operators configuring the opt-in overlay should use the literal form.
6. **GoReleaser auto-changelog excludes `docs:`/`test:`/`chore:` and the `feat:` subjects are fragmented.** Most of this arc's commits will NOT appear coherently in the auto-generated GitHub Release notes, and the 13 `feat:` subjects are granular. This curated migration note is the primary consumer release-note surface. This is expected per the `v0.6.0` precedent, not a defect.

## Rollback
A minor release is reversible by binary downgrade: reinstall the `v0.6.0` binary and `vh-agent-harness update` re-renders the `v0.6.0` corpus (the render is authoritative for `platform_managed` files). **Caveats:** rolling back (a) removes the new `sys-prompt` command surface, the embedded auto-gate system prompts, and the `doctor` `auto-classifier` check; (b) removes the F-intent permconfig transform seam, the dev-workflow staleness hardening, and the debate O5 refinement from the binary; (c) removes the `auto-classifier-pilot` pack from the embedded set (the v0.6.0 binary predates its promotion to `templates/overlays/`); a consumer who selected it by name must either drop it from `overlays:` or keep a project-local pack at `.vh-agent-harness/overlays/auto-classifier-pilot/` (which still resolves via the project-first seam) before re-rendering with the older binary — otherwise the name fails to resolve. The binary building blocks the overlay depends on are also gone, so enforcement goes inert until you return to `v0.7.0`. No data migration ran, so there is nothing to undo beyond the binary reinstall and re-render. A `local_adopted` customization survives the rollback and triggers the standard courtesy check on the way back up:
```bash
git checkout HEAD -- .opencode/         # only if an unrelated render diff is unwanted
```
