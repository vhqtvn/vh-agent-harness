## Watch-outs
1. **v0.13.0 has no GitHub Release page.** Tag `v0.13.0` (commit `552430b0`) was pushed to origin WITHOUT a published Release page — goreleaser refused on a dirty tree (`?? release-defer-evaluator.json` in the repo root). Consumers and operators should not expect a v0.13.0 Release page. This v0.13.1 release is the first in this line expected to publish a GitHub Release page end-to-end (Slice B of this arc fixes the dirty-tree cause).
2. **Push-only requires the annotated tag to already exist locally.** `scripts/release-tag.sh <v> --push-only` refuses with `tag <v> does not exist; cannot push-only (cut it first without --push-only)` if the tag has not been cut by the create flow first.
3. **Push-only requires an annotated tag object.** A lightweight tag (`git tag <v>` with no `-a`) is refused pre-push with `refs/tags/<v> is not an annotated tag object; push-only requires an annotated tag created by the full ceremony (got lightweight tag)`. This closes a contract gap caught by commit-reviewer on the inaugural pass (t1b-F1).
4. **Do not combine `--push-only` with override flags.** `--push-only` cannot be combined with `--override-release-version` / `--override-manifest-sha`; the DEFER gate is skipped in push-only mode, so there is nothing to override.
