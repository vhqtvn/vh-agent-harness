# Migration: v0.6.0

## Summary
- **Release class:** minor (semver compatible). No breaking changes; no profile/ownership-class change. The v0.5.0..HEAD arc carries five `feat:` commits (diagnostics-export, the `docs` verb, the typed-record DTO + append-only JSONL memory store with write-safety, and the releaser cut-time migration-note authoring flow) plus four `fix:` commits (the SECDATA-1 commit-gate traversal-uuid hardening and the commit-gate message-scratch reclaim/cleanup hardening) → MINOR bump; no breaking change → not v1.0.0. Last tag = v0.5.0; intended version = v0.6.0.
- **Upgrade path:** binary self-update, then re-render the corpus. Fully automatic — no manual file edits required for the default case.
- **Risk:** low. Six consumer-relevant changes in the arc, all non-breaking (one is a new operator command, one is a new command + doc-serving verb, one is an additive internal memory capability, one is an additive release-tooling capability, one is a security tightening, one is commit-gate cleanup hardening):
  - **diagnostics-export — new operator command** `vh-agent-harness diagnostics-export [--dry-run] [--output <path>]`. Packages harness state (`.opencode/state/`, `.local/coordinator/`, `.local/config/`, `docs/checkpoints/`) into a redacted, shareable `tar.gz` under repo-scoped `tmp/` (default `tmp/diagnostics-<timestamp>.tar.gz`). Redaction runs in three layers — field-name (`apikey`/`token`/`secret`/`password`/… → `***REDACTED(Nchars)***`, with subtree redaction under sensitive keys), whole-section (`secrets`/`env`/`credentials`/`models`), and value-pattern (Bearer tokens, AWS `AKIA` keys, `scheme://user:pass@host` connection strings). Binary/unparseable files are included as-is and flagged in the manifest for operator review. Never auto-uploads; the output path is symlink-escape-hardened (`resolveSymlinkSafe` + `assertRepoContained` + temp-file/`os.Rename` so the final component cannot be a symlink). Reimplemented in pure Go (no Node sidecar).
  - **`docs` verb — new command + doc-serving mechanism** `vh-agent-harness docs [key]`. Serves generic agent-workflow docs (the session/memory/prompt/skill model — identical for every adopter) from an embedded library (`templates/docs`) instead of installing them into your repo. With no argument, lists keys; with a KEY, prints that doc to stdout. The five current keys are `opencode-memory-model`, `opencode-prompt-guide`, `opencode-session-workflow`, `opencode-skills`, `temporary-files`. Resolution is override-first, embed-fallback: `.vh-agent-harness/docs-overrides.yml` maps a key to a live repo-relative file (this is how the repo dogfoods the library under continuous update — the installed binary reads live source, not a stale embed); adopters with no overrides file always get the embedded copy. The always-on `opencode-session-workflow` doc is materialized into `.vh-agent-harness/docs/` by install/update so opencode can load it via `instructions[]` (opencode loads files, not command output; it lives outside `.opencode/` so the drift scanner never flags it).
  - **typed-record DTO + append-only JSONL memory store with write-safety.** A new `internal/memory` layer: an 11-field typed-record DTO (`internal/memory/record`: `Type` persona/episodic/instruction, `Priority` low/normal/high/critical, `Scope` session/workstream; forward-compat decoding ignores unknown fields but rejects unknown enum values as corruption) and an append-only JSON Lines store (`internal/memory/store`) at two repo-relative paths — session `.opencode/state/sessions/<alias>/memory/records.jsonl` and workstream `.opencode/state/workstreams/<slug>/records.jsonl`. The reader is a bounded linear scan: skip-malformed (counted, never fatal), dedup by ID last-write-wins, AND-filter, sort by Priority then UpdatedAt, cap at `MaxRecords`; the write path guards `len(line) > maxLineBytes` before any filesystem side effect so the layer never writes data it cannot read back. Slice-3 write-safety hardens the append path with `flock(LOCK_EX)` → write → `fsync` data → `fsync` parent dir on create → unlock → close, plus a package-level per-path mutex to serialize same-process concurrent appends. **Purely additive:** the pre-existing flat memory files (`brief.md`, `resolved-context.md`, `open-questions.md`, `decision-log.md`) are never touched.
  - **releaser cut-time migration-note authoring (this very flow).** The releaser agent now digs the changes since the last tag, writes the canonical release migration note (`templates/migrations/v<next>.md` — this file), and delegates exactly one commit of that single file to the committer before the release-tag wrapper tags. Closes the gap where migration notes had to be authored manually at cut time. This is the first release cut under that flow. The releaser→committer edge follows the canonical gateExempt delegator contract (the committer, not the releaser, holds the gate).
  - **SECDATA-1 — commit-gate `_session_meta_path` traversal-uuid hardening (security fix).** `_session_meta_path()` interpolated a raw caller-controlled uuid into `${GATE_INDEX_DIR}/meta-${1}` with no validation; the result fed `rm -f` (arbitrary-file-deletion via the release no-lock branch), `cat` (read class), and a `python3` read+write block (arbitrary-file-overwrite via the heartbeat lock-free branch). Confirmed live: `release --uuid 'x/../../config'` deleted `.git/config` when `.git/commit-gate/meta-x/` existed. v0.6.0 adds a centralized `^[A-Za-z0-9_-]+$` uuid-shape guard at the single path-construction chokepoint so all call sites are protected uniformly; a non-conforming uuid echoes `""` and returns 0 (the reject signal under `set -euo pipefail`), making every downstream sink a safe no-op. Internal `_uuid` (`gen-uuid`, `[0-9a-f-]+`) is a strict charset subset, so the legit acquire→commit→release flow never trips. Two regression subtests pin both attack classes.
  - **commit-gate message-scratch reclaim/cleanup hardening.** The agent-owned commit-message scratch at `tmp/commit-gate-message/msg-${UUID}` was documented as "agent runs rm (optional best-effort)", but rm is in no agent's bash permission map — the cleanup was unenforceable and orphans had accumulated. The gate now owns that surface's reclamation: `_cleanup_own_scratch` reclaims the actual `--message-file` path directly on the commit/no-changes paths, `_gate_gc_sweep` sweeps aged `msg-*` orphans under `tmp/commit-gate-message/` with the same age-gate + protected-UUID skip used for `.git/commit-gate/`, and additional security chokepoints (`_safe_msg_reclaim_path`, the uuid-shape guard) protect the gate's privileged `rm`. A committed Go integration test guards the behavior. Agents MUST NOT `rm` under `.git/commit-gate/` or `tmp/commit-gate-message/` — the gate self-cleans both.

## Watch-outs
1. **diagnostics-export never auto-uploads.** The archive lands under repo-scoped `tmp/` (gitignored). Redaction is layered (field-name + whole-section + value-pattern), but it is defense-in-depth, not a guarantee — review the manifest's redaction counts and the flagged binary/unparseable files before sharing the bundle externally. If you carry secrets in non-standard field names or patterns the layers do not cover, they will not be redacted.
2. **`docs` override liveness.** If `.vh-agent-harness/docs-overrides.yml` maps a key to a deleted/moved file, `docs <key>` errors loudly rather than silently falling back to the embed (the operator asked for that specific file). Adopters with no overrides file always get the embedded copy; this liveness path is for the dogfood repo and projects that choose to override.
3. **typed-memory is additive-only.** Records persist in `records.jsonl` beside the pre-existing flat memory files; the flat files are never modified. Agents read typed records via the explicit-invocation seam documented in the memory-model doc (stable-vs-dynamic split, hard timeout with skip-never-block, code-point-safe character budget). Do not expect typed records to appear in agent context automatically — they are injected on explicit read.
4. **SECDATA-1 rejects traversal-uuids.** If any workflow was (ab)using a non-`[A-Za-z0-9_-]` `--uuid` against the commit-gate meta path (it should not have been — it is a security bug), it now no-ops safely. Legit uuids from `gen-uuid` (`[0-9a-f-]+`) never trip the guard.
5. **Commit-gate scratch is gate-owned now.** Agents MUST NOT `rm` under `.git/commit-gate/` or `tmp/commit-gate-message/` — the gate self-cleans both (immediate reclaim on commit/no-changes; aged orphan GC otherwise). `rm` was never in any agent's bash permission map; this just makes the documented cleanup actually enforceable.
6. **This is the first release cut under the cut-time note-authoring flow.** The migration note (`templates/migrations/v0.6.0.md`) is now authored by the releaser and committed by the committer before the tag, rather than authored manually at cut time. The note is embedded via `go:embed` and enforced by `TestMigrationNotes_Canonical`.
7. **GoReleaser auto-changelog excludes `docs:`/`test:`/`chore:` commits.** Seven `docs:` + one `test:` + one `chore:` commits in this arc will NOT appear in the auto-generated GitHub Release notes — only the `feat:`/`fix:` subjects survive. This curated migration note is the primary consumer release-note surface. This is expected, not a defect.

## Rollback
A minor release is reversible by binary downgrade: reinstall the v0.5.0 binary and `vh-agent-harness update` re-renders the v0.5.0 corpus (the render is authoritative for `platform_managed` files). **Caveats:** rolling back (a) re-introduces the SECDATA-1 traversal-uuid deletion/overwrite vector in `commit-gate.sh`; (b) reverts the message-scratch reclaim so `tmp/commit-gate-message/` orphans accumulate again; (c) removes the new `diagnostics-export` and `docs` command surfaces and the new `diagnostics-export` skill; (d) removes the typed-memory `internal/memory` package (existing `records.jsonl` files on disk are left in place — they are gitignored state, not corpus — but the binary no longer reads them); (e) removes the cut-time note-authoring flow from the release overlay. Do not roll back if the SECDATA-1 vector is a live concern. No data migration ran, so there is nothing to undo beyond the binary reinstall and re-render. A `local_adopted` customization of `commit-gate.sh` survives the rollback (it is preserved in both directions) and triggers the standard courtesy check on the way back up:
```bash
git checkout HEAD -- .opencode/         # only if an unrelated render diff is unwanted
```
