# Migration: v0.4.0

## Summary
- **Release class:** minor (semver compatible). No breaking change to the binary/roster/profile/ownership contract; two behavioral reversals of v0.3.0 surfaces and one new packaging-policy enforcement at the commit gate.
- **Upgrade path:** binary self-update, then re-render the corpus. Fully automatic for the default case.
- **Risk:** medium. Three consumer-visible net changes:
  - **New `exec-ro` read-only gate.** `vh-agent-harness exec-ro [--] <cmd>` is a general read-only execution gate enforced inside the Go binary. It is allowlisted in `opencode.jsonc` as `vh-agent-harness exec-ro *: allow` for every agent, so opencode never prompts for it — exec-ro is the sole gate for its allowance and is deny-only (executes the command exactly as given or DENIES; never asks, never rewrites). `GitMutationVerbs` is unified as a single Go source emitted to `allowed-commands.js` and consumed by both exec-ro's classifier and the shell-guard JS plugin, eliminating verb-list drift.
  - **Backlog model: W1 edit-blocking → hybrid split-commit + DEFER curation.** v0.3.0's W1 backlog-edit deny for `build`/`docs-steward` (permconfig object-form deny) is REMOVED — workers revert to flat `Edit=Allow` and can edit `docs/planning/backlog.md` directly again. In its place, the commit-gate `acquire` preflight now ENFORCES split-commit (a path list mixing `docs/planning/backlog.md` with any other path is rejected, exit nonzero, with a teaching recovery message). DEFER/p2 follow-up findings never become backlog rows directly: they land in `.local/coordinator/tasks/` as conditional candidates and reach `backlog.md` only after a trigger fires + promoter curation + a Definition of Ready. A new `backlog` skill and `check-defer-triggers.js` predicate checker ship with the model.
  - **shell-guard drops the command-rewrite.** v0.3.0's `b2a9808` rewrote global-flag-prefixed git commands in-place before the permission matcher; v0.4.0 removes the write-back (rewriting execution is unsafe for pipelines/subshells/redirects) and keeps `walkGitGlobals` verb classification for DECISIONS only (allow/deny/ask). Behavioral consequence: global-flag-prefixed git reads that relied on the rewrite (`git -C <repo-root> diff`, `git --no-pager log`) prompt again where no L2 allow rule matches; `--no-pager <readonly>` stays prompt-free via the L2 config rules (now the primary path).

## How to migrate (automated)
```bash
vh-agent-harness self-update            # pull the new binary (v0.4.0)
vh-agent-harness version                # expect: 0.4.0 (<label>)
vh-agent-harness update --dry-run       # ownership-safe preview; watch the exec-ro allowlist + commit-gate.sh + permconfig renders
vh-agent-harness update                 # applies platform_managed + active overlay_extension; ships exec-ro allowlist, the split-commit preflight, the reverted W1 deny, and the detect-only shell-guard
vh-agent-harness doctor                 # lint the result
```
No profile, ownership, or capability change is required. The `exec-ro` subcommand and its allowlist entry ship via `self-update` + re-render; the backlog model and the shell-guard rewrite removal ship by re-rendering the corpus. After `update`, `build`/`docs-steward` regain flat `Edit=Allow` (the v0.3.0 W1 deny is gone), and the commit-gate enforces split-commit on every `acquire`.

## What `update` handles for you
- Emits the `vh-agent-harness exec-ro *: allow` entry into every agent block in `opencode.jsonc` (exec-ro is the sole, deny-only gate for its allowance).
- Re-renders `.opencode/repo-configs/allowed-commands.js` and `.opencode/repo-configs/forbidden-patterns.core.js` so `GitMutationVerbs` is the single source consumed by both exec-ro's Go classifier and the shell-guard JS plugin.
- Re-renders `.opencode/scripts/commit-gate.sh` with the O1 split-commit preflight (rejects a path list mixing `docs/planning/backlog.md` with any other path; lexically normalizes each path first).
- Reverts the `build`/`docs-steward` edit decision to flat `Edit=Allow` (removes the v0.3.0 object-form backlog deny); every other agent's edit decision is byte-identical to before.
- Re-renders `.opencode/plugins/shell-guard-core.js` (plus the `shell-guard.js` shim and `shell-guard/eval.js` bridge) WITHOUT the rewrite — detect/parse only.
- Ships the `backlog` skill (`.opencode/skills/backlog/SKILL.md`) and `check-defer-triggers.js`; reconciles AGENTS.md and `docs/coordination/` to the hybrid split-commit + DEFER + eventual-consistency model.
- Reconciles schema on seeded files; preserves everything tagged `local_adopted` / inactive `overlay_extension` unchanged.

## Watch-outs
1. **shell-guard no longer rewrites global-flag-prefixed git reads.** v0.3.0's in-place rewrite is gone (rewriting execution is unsafe). Operators and agents may see permission prompts RETURN for global-flag-prefixed git read commands (`git -C <repo-root> diff`, `git -p log`, `git --paginate log`) where no L2 allow rule matches. `--no-pager <readonly>` stays prompt-free via the L2 config rules (now the primary path). To keep a specific global-flag read prompt-free, add an explicit `git -C <cwd> *` / `git --no-pager *` L2 allow rule, or route the read through the new `vh-agent-harness exec-ro -- git …` gate. The mutation-slip guard still DENIES `git … <mutation>` regardless of flag count.
2. **Split-commit is now ENFORCED at the commit-gate preflight.** A path list passed to `commit-gate.sh acquire` (or the committer agent) that contains `docs/planning/backlog.md` alongside any other path is REJECTED with exit nonzero and a teaching recovery message — split-commit is no longer advisory. Commit code first (without the ledger), then re-read `docs/planning/backlog.md` from disk, edit only your rows, run `node .opencode/scripts/normalize-backlog.js --check`, and commit the backlog alone. Paths are lexically normalized first, so `./docs/planning/backlog.md` or `docs/./planning/backlog.md` cannot bypass the guard. Do NOT `commit-gate.sh revert docs/planning/backlog.md` to resolve a conflict — re-read + re-apply instead.
3. **The v0.3.0 W1 backlog-edit deny is reverted.** `build`/`docs-steward` can now directly edit `docs/planning/backlog.md` again (flat `Edit=Allow`). Conflict safety moved to the gate (Watch-out 2), not the edit layer. If you had local tooling that assumed the deny, update it to expect the split-commit enforcement instead. The `edit-guard.js` plugin from the development arc never shipped — it was added and deleted within the same arc (see Non-consumer changes).
4. **`exec-ro` is deny-only.** Because it is allowlisted in `opencode.jsonc`, opencode NEVER prompts for it; exec-ro itself is the only gate and hard-denies anything it cannot prove read-only. A command exec-ro refuses will not fall back to a prompt — it fails. Use plain `vh-agent-harness exec <cmd>` (subject to the normal permission matcher) for anything exec-ro's allowlist does not cover, or add an explicit allow rule.
5. **Customized shell-guard.** If a previous `local_adopted` edit customized `.opencode/plugins/shell-guard-core.js`, the migration courtesy check fires and `update` will NOT silently clobber it. Review the diff and re-merge your customization onto the canonical detect/parse-only form so the rewrite removal lands.

## Rollback
A minor release is reversible by binary downgrade: reinstall the v0.3.0 binary and `vh-agent-harness update` re-renders the v0.3.0 corpus (the render is authoritative for `platform_managed` files). Rolling back re-imposes the v0.3.0 W1 backlog-edit deny for `build`/`docs-steward` (workers lose direct backlog edits again), re-adds the shell-guard command-rewrite (global-flag-prefixed git reads become prompt-free again via the rewrite), removes the `exec-ro` subcommand and its allowlist entry, removes the commit-gate split-commit preflight (backlog may again travel in a mixed commit), and removes the `backlog` skill, `check-defer-triggers.js`, and the DEFER-curation routing. No data migration ran, so there is nothing to undo beyond the binary reinstall and re-render. A `local_adopted` customization survives the rollback (it is preserved in both directions):
```bash
git checkout HEAD -- .opencode/         # only if an unrelated render diff is unwanted
```

## Non-consumer changes
The three consumer-visible surfaces landed across commits `601cb51` (exec-ro — general read-only gate + single-source git-verb feed), `2ac2f41` (shell-guard — drop command-rewrite, keep detect/parse for decisions), `b56b2bd` (backlog — unwind W1 edit-blocking, ship hybrid split-commit + DEFER curation), and `77dc6f4` (backlog — eventual-consistency: gate split-preflight O1 + promoter docs O2 + de-authorize backlog-reminder O4). The `5d244a1` `feat(edit-guard)` commit (curated W1 denial of `backlog.md` edits) and its companion `b2f6504` (test pinning edit-guard's `PROTECTED_PATH`) were INTERMEDIATE steps in the same arc that were unwound by `b56b2bd` — `edit-guard.js` was added in `5d244a1` and deleted in `b56b2bd`, so it never ships in v0.4.0 and is not a consumer-visible surface; it is listed here only so the arc's commit trail is traceable. Likewise the `backlog-reminder.js` plugin added in `b56b2bd` was de-authorized and deleted in `77dc6f4` (harvesting `console.error` was too weak to nudge agents reliably) and does not ship. Supporting docs-only work (`5d911ba` backlog follow-up rows, `f85e067` researches source packet, `65066dd` tooling ignore-negation) and the test suites (`internal/execro/classifier_test.go`, `internal/cli/commit_gate_preflight_test.go`, updated `internal/permission/shellguard_hook_test.go`) are internal-only and need no consumer action beyond this note. README.agent.md's exec-ro and "Track work" sections were reconciled to the shipped model in the same arc.
