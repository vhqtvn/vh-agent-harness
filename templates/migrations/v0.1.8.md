# Migration: v0.1.8

## Summary
- **Release class:** patch (semver compatible). No breaking changes.
- **Upgrade path:** binary self-update, then re-render the corpus. Fully automatic — no manual file edits required.
- **Risk:** low. Affects rendered `.opencode/` config only; no domain/runtime contract change.

## What changed (consumer-visible only)
| area | change | ships-via | class |
|------|--------|-----------|-------|
| shell-guard | `git-mutation-bypass` allowIf gains a command-position-anchored inspector carve-out (echo/cat/printf/grep); `printf` added to the sanctioned allowlist | `update` (core template) | non-breaking |
| gated-commit | `commit-gate.sh` self-cleans orphan scratch on the `no_changes` path; SKILL.md step 5 drops the un-followable manual `rm` | `update` (core template) | non-breaking |
| commit-guide | Builder-side rule: hand committer only this session's files; sanctioned unblock is `commit-gate.sh revert <paths>` | `update` (rendered doc) | docs-only |

## How to migrate (automated)
```bash
vh-agent-harness self-update            # pull the new binary
vh-agent-harness version                # expect: 0.1.8 (<label>)
vh-agent-harness update --dry-run       # ownership-safe preview
vh-agent-harness update                 # applies platform_managed + active overlay_extension
vh-agent-harness doctor                 # lint the result
```

## What `update` handles for you
- Overwrites `platform_managed` files with the new shell-guard rule + sanctioned allowlist (printf added).
- Re-renders `commit-gate.sh` and the gated-commit `SKILL.md` with the self-cleaning scratch behavior.
- Reconciles schema on seeded files; preserves everything tagged `local_adapted` / inactive `overlay_extension` unchanged.

## Watch-outs
1. **Customized `allowed-commands.js`** — if a previous `local_adopted` edit customized it, the existing migration courtesy check fires and `update` will NOT silently clobber it. Review the diff and re-merge your customization onto the canonical form if you want the new `printf` entry.
2. **Orphan scratch under `.git/commit-gate/`** — old `msg-*` / `paths-*` files left by pre-0.1.8 flows are swept on the next commit/release/no_changes path (TTL-gated, default 3600s). No manual cleanup needed.

## Verification commands
```bash
vh-agent-harness doctor                 # expect: no new findings
vh-agent-harness diff                   # expect: clean (no drift vs rendered)
git status                              # expect: only .opencode/ renders changed
```

## Rollback
Patch release is reversible: reinstall the v0.1.7 binary and `vh-agent-harness update` re-renders the prior corpus. No data migration ran, so there is nothing to undo beyond the rendered-tree revert:
```bash
git checkout HEAD -- .opencode/         # if the render diff is unwanted
```
